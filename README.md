# Biligumi

一个面向哔哩哔哩番剧页的浏览器插件，用于把当前播放页与 Bangumi（`bgm.tv`）的作品资料快速衔接起来。插件提供两种主要能力：一是在番剧播放页自动弹出悬浮资料窗，二是在扩展弹窗中基于网页标题做分词辅助检索，方便快速查找作品、Staff、角色、声优与常用标签等信息。

- 在哔哩哔哩番剧播放页自动识别当前作品标题
- 自动请求 Bangumi API 并展示关联动画 / 三次元节目资料
- 使用悬浮窗展示核心 Staff、主要角色与声优、常用标签、评分与年份
- 支持折叠/展开悬浮窗，减少观看时的干扰
- 提供“切换番剧时自动展开悬浮窗”的本地设置
- 在扩展弹窗中读取当前页面标题并自动分词
- 支持点击分词结果快速拼接搜索关键词
- 支持手动检索作品并查看关联条目
- 通过后台脚本代理 API 请求，降低内容脚本直接请求带来的限制与异常影响

## 适用场景

如果你经常在哔哩哔哩看番，同时又想快速查看作品在 Bangumi 上的条目信息，这个插件适合你。它尤其适合以下场景：

- 想快速查看作品的 Bangumi 评分
- 想快速确认一部作品的导演、脚本、角色设计、音乐等核心 Staff
- 想查看主要角色及其声优
- 想对比续作、前传、关联作品
- 想从哔哩哔哩页面标题中快速提取关键词去 Bangumi 检索

## 效果说明

插件主要包含两个界面：

### 1. 番剧页悬浮窗

当你进入哔哩哔哩番剧播放页时，插件会监听页面地址变化，在识别到新番剧页面后自动读取标题并查询 Bangumi 数据。随后页面右下角会出现悬浮资料窗，展示：

- 当前匹配或关联作品
- Bangumi 评分
- 中文名 / 原名
- 上映年份
- 核心 Staff
- 主要角色与声优
- 常用标签
- 跳转到 Bangumi 详情页的链接

### 2. 扩展弹窗检索面板

点击浏览器工具栏中的插件图标后，会打开一个弹窗面板。它会尝试从当前标签页标题中提取中、日、英文字词，用户可以：

- 点击提取出的词语，自动加入搜索框
- 手动修改搜索词
- 按回车或点击按钮发起搜索
- 查看匹配作品及其关联资料
- 控制“切换番剧时悬浮窗自动展开”开关

## 安装方式

目前项目结构看起来是一个未经过打包的原生浏览器扩展，适合以“加载已解压的扩展程序”方式安装。

### Chrome / Edge

1. 下载仓库 zip 压缩包并解压为文件夹
3. 打开浏览器扩展管理页：
   - Chrome：`chrome://extensions/`
   - Edge：`edge://extensions/`
4. 打开“开发者模式”
5. 选择“加载已解压的扩展程序”
6. 选择项目根目录
7. （可选）安装完成后，把插件固定到工具栏，便于使用

## 使用方法

### 自动悬浮窗模式

1. 打开任意哔哩哔哩番剧播放页，例如：
   - `https://www.bilibili.com/bangumi/play/...`
2. 等待插件自动识别页面标题
3. 插件会在页面右下角弹出 Bangumi 资料悬浮窗
4. 可点击“折叠”收起悬浮窗，再通过悬浮按钮重新展开

### 手动检索模式

1. 点击浏览器工具栏中的插件图标
2. 在弹窗中查看自动提取的标题分词
3. 点击分词把词语加入搜索框，或直接手动输入关键词
4. 按回车或点击“检索”按钮
5. 查看返回的作品条目、Staff、角色、声优与标签信息

## 使用示例

### 示例一：在播放页自动查看作品资料

假设你正在哔哩哔哩播放页观看某部番剧，插件会从页面标题中提取作品名，并自动去 Bangumi 搜索。若命中条目，你可以直接看到该作品的：

- 导演 / 监督
- 系列构成 / 脚本
- 角色设计 / 作画监督
- 主要角色与对应声优
- 标签与评分

并且可以直接跳转到对应的 Bangumi 页面
这适合在追番时快速补充背景资料，而不用手动切到 Bangumi 搜索。

### 示例二：用分词辅助检索非官方番剧作品

在个人上传的非官方番剧视频中，点击插件图标，插件弹窗会先把标题拆成多个可点击词语，你可以只挑作品核心词进行组合搜索，从而提高检索成功率。

例如，你可以：

- 点击选主标题词
- 发起搜索

这样比直接复制整个标题更容易命中正确条目。

## 技术实现概览

项目当前是一个浏览器扩展，核心由以下部分组成：

- `manifest.json`：扩展声明文件，定义权限、图标、后台脚本、弹窗页面和内容脚本
- `content.js`：运行在哔哩哔哩番剧页面中的内容脚本，负责监听页面变化、抽取标题、创建悬浮窗、渲染 Bangumi 数据
- `popup.html`：扩展弹窗页面结构与样式
- `popup.js`：扩展弹窗逻辑，包括标题分词、手动检索、设置项存储
- `background.js`：后台 Service Worker，负责代理对 Bangumi API 的请求

插件当前依赖的浏览器能力主要包括：

- `activeTab`
- `tabs`
- `storage`

访问域名权限包括：

- `*.bilibili.com`
- `api.bgm.tv`

## 数据来源

本插件目前通过 Bangumi 公开接口获取作品信息，主要涉及：

- 作品搜索
- 作品详情
- 关联作品
- 参与人员（Staff）
- 角色与声优信息

相关请求由扩展后台脚本统一发起，再返回给内容脚本或弹窗脚本处理。



### 兼容性说明

从当前实现看，项目主要面向 Chromium 内核浏览器，包括但不限于：

- Google Chrome
- Microsoft Edge
- 其他兼容 Manifest V3 的 Chromium 浏览器

## 已知限制

- 当前匹配逻辑主要依赖页面标题清洗与 Bangumi 搜索结果，遇到别名、译名差异较大的作品时可能命中不够稳定
- 搜索结果依赖 Bangumi 搜索逻辑（会自动爬取靠前的搜索结果），如果B站翻译或番剧名称与 Bangumi 不一致，或者由于 Bangumi 搜索机制的问题，可能会导致搜索到无关条目
